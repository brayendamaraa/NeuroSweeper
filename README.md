# NeuroSweeper

NeuroSweeper is a neural network–based Minesweeper agent trained from data generated by a deterministic solver.  
The goal of the project is to explore whether a convolutional neural network can learn Minesweeper decision patterns from gameplay data.

The training data is generated automatically by a deterministic Minesweeper agent that uses brute-force reasoning and simple logical rules. The resulting dataset is then used to train a neural network that predicts the best action for a given board state.

---

# Project Pipeline

The NeuroSweeper system consists of four main components:

1. Data Generation  
2. Data Encoding  
3. Model Architecture  
4. Model Evaluation

---

# 1. Data Generation

The dataset is generated by simulating Minesweeper games using a **deterministic agent**.

The deterministic agent solves the board using:

- Basic Minesweeper rules
- Brute-force reasoning when simple rules are insufficient

Examples of the basic rules used include:

- If a revealed cell has the same number of unrevealed neighbors as its number, all neighbors are mines.
- If the number of flagged neighbors equals the number on the cell, the remaining neighbors are safe.

During gameplay, the agent records:

- **State Matrix** → the visible Minesweeper board state  
- **Action Matrix** → the action taken by the agent at that state

These pairs form a **supervised learning dataset** used to train the neural network.

The Minesweeper environment is implemented in `game_simulate.py`, which supports beginner, intermediate, and expert difficulties.

# 2. Data Encoding

Neural networks require numerical tensors as input. Therefore the Minesweeper state and actions must be encoded into matrices.

## State Encoding

Each board cell can be in one of several states:

| Cell Type | Value |
|----------|------|
| Number (0–8) | Revealed neighbor mine count |
| Unrevealed | 9 |
| Flagged | 10 |
